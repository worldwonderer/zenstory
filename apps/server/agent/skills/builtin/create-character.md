# 创建角色

帮助用户创建小说或剧本中的角色，包括外貌、性格、背景等设定。

## Triggers
- 创建角色
- 新建角色
- 添加角色
- 设计角色
- 角色设定

## Instructions
你现在进入角色创建模式。请帮助用户创建一个完整的角色设定。

### 任务目标
创建一个立体、鲜活的角色，包含完整的外貌、性格、背景设定，使其能够在故事中自然地行动和发展。

### 执行流程

1. **收集信息**：
   - 询问角色在故事中的定位（主角/配角/反派）
   - 了解用户对角色的初步构想
   - 确认角色所处的世界观背景

2. **设计基本信息**：
   - 姓名（考虑时代背景和文化特征）
   - 年龄、性别、身份职业
   - 外貌特征（抓住2-3个标志性特点）

3. **塑造性格层次**：
   - 表面性格（他人眼中的形象）
   - 内在性格（真实的自我）
   - 性格缺陷（让角色更真实）
   - 口头禅或习惯动作

4. **构建背景故事**：
   - 成长经历和关键事件
   - 形成当前性格的原因
   - 内心的渴望和恐惧

5. **建立人物关系**：
   - 与其他角色的关系
   - 关系中的冲突点

### 输出格式

使用 `create_file` 工具创建角色文件，文件类型为 `character`，格式如下：

```
# [角色姓名]

## 基本信息
- 年龄：
- 性别：
- 身份：

## 外貌特征
（2-3段描写，突出标志性特点）

## 性格特点
- 表面性格：
- 内在性格：
- 性格缺陷：
- 口头禅/习惯：

## 背景故事
（角色的过往经历，解释性格成因）

## 人物关系
（与其他角色的关系）

## 角色弧光
（角色在故事中可能的成长方向）
```

### 质量标准

- 角色要有内在逻辑，行为动机要合理
- 性格要有层次感，避免非黑即白的脸谱化
- 背景故事要能支撑角色的性格和行为
- 要有明显的辨识度，让读者能记住
- 预留成长空间，角色可以在故事中变化

### 人物关系动态网

**关系圈层设计**：
- 亲密圈（3人）：家人/爱人/挚友
- 利益圈（4人）：盟友/下属/合作者/资源方
- 对立圈（5人）：宿敌/竞争者/立场相悖者/潜在威胁/隐藏敌人

**关系变化节奏**：
- 关系升温：敌人→中立→朋友→生死之交
- 关系降温：误会/背叛/立场分歧导致恶化
- 关系反转：盟友变敌人、敌人变盟友
- 关系考验：危机时刻检验关系真伪

**关系冲突设计**：
- 利益冲突：资源争夺、目标对立
- 情感冲突：爱恨纠葛、信任危机
- 立场冲突：价值观碰撞、阵营对立

### 人物反差点

**反差设计原则**：
- 外表与内心的反差（冷酷外表/温柔内心）
- 能力与性格的反差（强大实力/社恐性格）
- 身份与行为的反差（高贵身份/接地气行为）

**热门人设参考**：
- 扮猪吃虎型：看似弱小实则强大
- 反差萌型：严肃外表下的可爱一面
- 成长型：从弱小到强大的蜕变

### 上下文利用

- 参考项目中的世界观设定，确保角色符合时代背景
- 参考已有角色，设计差异化的性格和定位
- 参考大纲文件，确保角色能服务于故事主线
