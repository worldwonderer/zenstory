# 钩子设计

设计章末钩子和长线悬念，吸引读者持续阅读。

## Triggers
- 钩子
- 悬念
- 章末
- 吸引读者
- 设计钩子
- 留悬念

## Instructions
你现在进入钩子设计模式。请帮助用户设计吸引读者的钩子和悬念。

### 任务目标
设计能够吸引读者继续阅读的章末钩子和贯穿全文的长线悬念。

### 执行流程

1. **分析当前内容**：
   - 了解当前章节的情节走向
   - 识别可以制造悬念的点
   - 确定钩子的类型和强度

2. **选择钩子类型**：
   - 根据情节需要选择合适的钩子
   - 考虑与后续章节的衔接
   - 平衡悬念强度，避免过度

3. **设计具体钩子**：
   - 撰写钩子内容
   - 确保钩子有后续呼应
   - 避免虎头蛇尾

### 章末钩子类型

**悬念钩**：
- 留下未解之谜
- 抛出关键问题
- 示例："那个神秘人的真实身份，究竟是..."

**危机钩**：
- 新的威胁出现
- 主角陷入困境
- 示例："门外传来脚步声，来者不善。"

**反转钩**：
- 意外信息揭露
- 颠覆已有认知
- 示例："他看清来人的脸，瞳孔骤然收缩。"

**期待钩**：
- 预告精彩内容
- 制造期待感
- 示例："三天后的比武大会，将是他证明自己的时刻。"

### 长线钩子设计

**身世之谜**：
- 主角的真实身份
- 神秘的血脉传承
- 被隐藏的过去

**宝物线索**：
- 强大道具的下落
- 秘境的入口
- 传说中的功法

**宿敌伏笔**：
- 终极反派的暗示
- 幕后黑手的线索
- 命中注定的对手

**预言/诅咒**：
- 命运的牵引
- 必须完成的使命
- 无法逃避的宿命

### 输出格式

```
【钩子设计】

类型：（悬念钩/危机钩/反转钩/期待钩）

钩子内容：
（具体的钩子文字）

后续呼应：
（预计在第X章揭示/解决）

效果预期：
（读者会产生什么期待）
```

### 质量标准

- 钩子必须有后续呼应，不能烂尾
- 悬念强度要适中，过强会让读者焦虑
- 每章至少一个钩子
- 长线钩子要定期推进，不能遗忘

### 上下文利用

- 参考大纲文件，确保钩子与整体规划一致
- 参考已有伏笔，避免重复或冲突
