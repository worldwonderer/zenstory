# 对话生成

生成符合角色性格的对话，让人物通过语言展现个性。

## Triggers
- 写对话
- 对话
- 对白
- 台词
- 人物对话

## Instructions
你现在进入对话生成模式。请帮助用户创作生动的人物对话。

### 任务目标
创作符合角色性格、推动情节发展的对话内容。

### 执行流程

1. **了解场景**：
   - 对话发生的场景和背景
   - 参与对话的角色
   - 对话的目的（冲突/和解/信息传递等）

2. **分析角色**：
   - 查阅角色设定文件
   - 把握每个角色的说话风格
   - 注意角色间的关系和态度

3. **设计对话**：
   - 每句话都要有目的
   - 通过对话展现性格差异
   - 加入适当的动作和神态描写

### 输出格式

```
"对话内容，"角色A说道，动作/神态描写。

角色B的反应描写。"回应内容。"
```

使用 `edit_file` 工具将对话插入到指定位置。

### 质量标准

- 每个角色的语言风格要有辨识度
- 对话要推动情节或揭示性格
- 避免无意义的寒暄和废话
- 潜台词比直白表达更有张力

### 对话进阶技巧

**潜台词设计**：
- 表面说A，实际表达B
- 通过语气、停顿暗示真实意图
- 让读者"听懂"角色没说出口的话

**信息差运用**：
- 角色A知道而B不知道的信息
- 读者知道而角色不知道的信息
- 利用信息差制造张力和期待

**对话功能**：
- 推进情节：通过对话揭示新信息
- 展现性格：语言风格体现人物特点
- 制造冲突：观点碰撞、立场对立
- 埋设伏笔：看似无意的话语暗藏玄机

### 对话禁忌

**避免无意义寒暄**：
- 删除"你好"、"再见"等无信息量对话
- 每句对话都要有目的

**避免直白表达**：
- 不要让角色直接说出内心想法
- 用行动和暗示代替直白陈述

**避免信息重复**：
- 不要通过对话重复读者已知的信息
- 对话要推进而非回顾

**避免角色同质化**：
- 不同角色说话方式要有区别
- 注意用词、句式、语气的差异

### 上下文利用

- 必须参考角色设定文件中的性格和口头禅
- 参考角色关系，体现在称呼和态度上
