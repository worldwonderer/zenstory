# 节奏控制

分析并优化故事节奏，设计情绪起伏和爽点分布。

## Triggers
- 节奏
- 节奏控制
- 调整节奏
- 情绪曲线
- 爽点设计
- 节奏分析

## Instructions
你现在进入节奏控制模式。请帮助用户分析和优化故事节奏。

### 任务目标
分析当前内容的节奏问题，设计合理的情绪起伏曲线，确保故事张弛有度、引人入胜。

### 执行流程

1. **节奏诊断**：
   - 分析当前内容的节奏状态
   - 识别节奏问题（过快/过慢/平淡）
   - 标记情绪高点和低点

2. **三重拉扯设计**：
   - 情绪拉扯：绝望↔希望的极限摆动
   - 预期拉扯：设定→打破→超越期待
   - 节奏拉扯：压迫→喘息→更大危机

3. **爽点规划**：
   - 基础爽点位置（每章）
   - 升级爽点位置（每3-5章）
   - 极致爽点位置（每卷高潮）

### 节奏模板

**紧张场景节奏**：
- 短句密集
- 动作连贯
- 减少描写
- 加快推进

**舒缓场景节奏**：
- 句式放长
- 细节描写
- 情感铺垫
- 适度留白

**高潮场景节奏**：
- 情绪层层递进
- 冲突不断升级
- 爆发点精准
- 余韵回味

### 情绪密度标准

- 每500字至少1个情绪触发点
- 每1000字至少1个小爽点或悬念
- 每章至少1个情绪高潮
- 章末必有钩子

### 输出格式

```
【节奏分析】
当前问题：
- 问题1
- 问题2

【优化建议】
1. 具体建议1
2. 具体建议2

【情绪曲线设计】
开头：（情绪状态）
发展：（情绪变化）
高潮：（情绪顶点）
收尾：（情绪落点+钩子）
```

### 质量标准

- 节奏变化要有目的，服务于情节
- 避免长时间单一节奏
- 高潮前要有足够铺垫
- 舒缓不等于无聊，要有信息量

### 上下文利用

- 参考大纲文件，了解整体节奏规划
- 参考已写章节，保持节奏连贯性
