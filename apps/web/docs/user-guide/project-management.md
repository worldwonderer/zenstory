# 项目管理

zenstory 以项目为核心单位组织您的创作内容。每个项目都是一个独立的工作空间,拥有完整的文件系统、AI 对话记录和版本历史。本文档将详细介绍如何高效地管理您的写作项目。

## 项目列表

![所有项目页面，展示项目列表与「新建项目」按钮](/docs-images/user-guide/project-management.png)

项目切换器位于顶部工具栏左侧,点击当前项目名称即可打开项目列表。这里展示了您可以访问的所有项目。

### 查看所有项目

项目列表采用下拉菜单设计,包含以下信息:

- **项目名称** - 项目的标题
- **项目描述** - 简短的项目说明(可选)
- **当前项目标识** - 当前选中的项目会显示对勾图标

列表中的项目按照最近访问时间排序,最近打开的项目显示在顶部。

### 搜索项目

当您拥有多个项目时,可以使用搜索功能快速定位:

1. 点击项目切换器打开下拉菜单
2. 在搜索框中输入项目名称的关键词
3. 列表会实时过滤,只显示匹配的项目

**搜索特点**:
- 支持模糊匹配
- 不区分大小写
- 实时过滤结果

## 创建项目

创建新项目非常简单,您可以从项目切换器中快速创建。

### 快速创建步骤

1. 点击项目切换器打开下拉菜单
2. 点击底部的 **"+ 创建项目"** 按钮
3. 输入新项目的名称
4. 点击 **"创建"** 按钮或按 **Enter** 键

创建完成后,系统会自动切换到新项目,您就可以开始添加文件了。

### 项目初始化

新创建的项目会自动包含以下默认文件夹结构(中文环境):

```
我的小说/
├── 大纲/           # 存放故事大纲
├── 正文/           # 存放手稿内容
├── 角色/           # 存放角色设定
└── 设定/           # 存放世界观
```

这些文件夹根据项目模板自动生成,帮助您快速开始创作。

### 项目命名建议

为了方便管理和识别,建议您在命名项目时遵循以下原则:

- **明确具体** - 使用小说名称或项目代号
- **简洁易记** - 避免过长的项目名称
- **区分系列** - 如果有多个相关项目,使用系列名+序号的方式

**示例**:
- ✅ 《星际迷途》第一部
- ✅ 科幻短篇集 2024
- ❌ 新项目1
- ❌ 未命名项目

> **参考**: 详细的首次项目创建流程,请参阅 [创建第一个项目](../getting-started/first-project.md)

## 切换项目

切换项目让您可以在不同的创作项目之间快速跳转,每个项目的文件和对话记录都保持独立。

### 如何切换项目

1. 点击顶部工具栏的项目切换器
2. 在项目列表中找到目标项目
3. 点击项目名称即可切换

切换后:
- 左侧文件树会显示新项目的文件结构
- AI 助手会加载新项目的对话历史
- 编辑器会显示新项目的最后打开的文件
- 浏览器地址栏会更新为新项目的 URL

### 快速跳转

**通过 URL 直接访问**:

每个项目都有独立的 URL 地址:
```
https://your-domain.com/project/{project-id}
```

您可以:
- 将常用项目的 URL 添加到书签
- 分享项目链接给协作者(如果有协作功能)
- 使用浏览器的前进/后退按钮在项目间导航

### 项目切换注意事项

- 切换项目会自动保存当前编辑器的内容
- AI 对话记录独立保存在各个项目中
- 文件树的展开/折叠状态会被记忆
- 每个项目有独立的版本历史记录

## 编辑项目信息

您可以根据需要随时修改项目的基本信息,包括项目名称和描述。

### 修改项目名称

1. 打开项目切换器
2. 将鼠标悬停在要编辑的项目上
3. 点击出现的 **铅笔图标**(编辑按钮)
4. 在内联输入框中输入新的项目名称
5. 按 **Enter** 保存,或按 **Esc** 取消

**编辑提示**:
- 鼠标悬停时编辑图标才会显示
- 编辑输入框会自动聚焦并选中当前名称
- 支持快捷键操作
- 点击输入框外部会自动保存

### 修改项目描述

目前项目描述需要通过项目的元数据设置来修改。后续版本将在项目切换器中提供直接编辑功能。

### 编辑时的限制

- 无法编辑正在编辑中的项目名称(需先完成或取消当前编辑)
- 项目名称不能为空
- 建议使用简洁明了的名称

## 删除项目

当您不再需要某个项目时,可以将其删除。**请注意,删除操作是永久性的,删除后的数据无法恢复。**

### 删除步骤

1. 打开项目切换器
2. 将鼠标悬停在要删除的项目上
3. 点击出现的 **垃圾桶图标**(删除按钮)
4. 在弹出的确认对话框中点击 **"确认"**

**删除前的保护机制**:

系统会在以下情况下阻止删除操作:

- **最后一个项目** - 系统要求至少保留一个项目,无法删除当前唯一的项目
- **确认对话框** - 必须明确确认才会执行删除

### 删除后的影响

删除项目会永久移除:

- ✗ 项目的所有文件(大纲、手稿、角色卡、世界观)
- ✗ 所有的 AI 对话记录
- ✗ 所有的版本历史和快照
- ✗ 项目的元数据设置

**重要提示**:
- 删除操作不可撤销
- 数据无法恢复
- 建议在删除前先导出重要内容

### 安全删除建议

在删除项目之前,建议您:

1. **导出重要内容** - 使用导出功能备份关键文件
2. **确认不再需要** - 仔细检查项目中是否有重要资料
3. **保留版本快照** - 如果可能,先创建项目的完整快照

## 项目间的关系

理解项目之间的独立性有助于您更好地组织创作内容。

### 每个项目完全独立

zenstory 的项目采用完全隔离的设计:

**独立的内容空间**:
- 每个项目拥有独立的文件树
- 文件和文件夹在不同项目间不共享
- 角色卡、世界观设定仅在当前项目内有效

**独立的 AI 上下文**:
- AI 助手对每个项目有独立的理解和记忆
- 在一个项目中训练的 AI 不会影响其他项目
- 对话历史完全隔离

**独立的版本历史**:
- 每个文件有独立的版本记录
- 快照功能仅在当前项目内有效
- 版本回滚不影响其他项目

### 文件不跨项目共享

目前zenstory 不支持跨项目共享文件。如果您需要在多个项目中使用相同的设定,有以下替代方案:

**方案 1: 复制内容**
1. 在源项目中打开文件
2. 复制文件内容
3. 在目标项目中创建新文件并粘贴

**方案 2: 模板复用**
1. 在源项目中整理好设定结构
2. 使用导出功能导出
3. 在新项目中参考导出的内容重新创建

**未来规划**:
- 素材库功能将支持跨项目共享参考素材
- 项目模板功能将支持从现有项目创建新项目

### 项目适用场景

根据项目的独立性特点,建议您按以下方式组织项目:

**一个项目 = 一部作品**:
- 📚 一部长篇小说
- 📚 一个短篇小说集
- 📚 一个系列故事

**不建议的做法**:
- ❌ 在一个项目中混杂多部不相关的作品
- ❌ 将一个作品拆分到多个项目中

### 项目组织的最佳实践

**清晰的项目结构**:
```
项目名称
├── 大纲/           # 故事架构
├── 正文/           # 章节内容
│   ├── 第一章
│   ├── 第二章
│   └── ...
├── 角色/           # 人物设定
│   ├── 主角
│   ├── 配角A
│   └── ...
└── 设定/           # 世界观
    ├── 力量体系
    ├── 势力分布
    └── ...
```

**利用 AI 理解上下文**:
- 将所有相关内容放在同一项目中
- AI 会自动关联角色、世界观和情节
- 保持项目内容的完整性和一致性

## 小贴士

1. **项目命名的规范性** - 使用清晰的命名规则,方便在多个项目间快速识别。

2. **定期清理不再需要的项目** - 删除确定不再使用的项目,保持项目列表的整洁。

3. **重要项目定期导出** - 养成定期导出重要项目内容的习惯,防止意外丢失。

4. **利用项目独立性** - 大胆尝试不同的创作方向,每个项目都是独立的实验空间。

5. **项目描述的使用** - 为项目添加简短描述,帮助您快速回忆项目的主题和进展。

6. **避免过度细分** - 一部完整的作品建议放在一个项目中,便于 AI 理解整体上下文。

## 常见问题

**Q: 一个账户可以创建多少个项目?**
A: 目前没有项目数量限制,您可以根据需要创建任意数量的项目。

**Q: 可以在不同设备间同步项目吗?**
A: 是的,项目数据存储在云端,您在不同设备上登录后会自动同步所有项目。

**Q: 删除的项目可以恢复吗?**
A: 很遗憾,删除操作是永久性的,删除后的项目数据无法恢复。删除前请务必确认。

**Q: 为什么无法删除最后一个项目?**
A: 系统要求至少保留一个项目,这是为了确保您始终有一个可工作的项目空间。如果需要清空内容,可以删除项目内的文件而不是删除整个项目。

**Q: 可以为项目设置封面或图标吗?**
A: 目前暂不支持项目封面和自定义图标功能,后续版本会考虑添加此功能。

**Q: 项目之间可以合并吗?**
A: 目前不支持直接合并项目。您可以通过复制文件内容的方式手动合并,但需要注意 AI 的上下文不会随之迁移。

**Q: 可以与他人共享项目吗?**
A: 目前的版本主要面向个人使用,协作共享功能正在规划中,敬请期待后续更新。

**Q: 项目的 AI 记忆会占用存储空间吗?**
A: AI 记忆存储在服务器端,不会占用您的本地存储空间。每个项目的 AI 上下文数据由系统自动管理。
