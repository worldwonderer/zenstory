## 1. 背景与目标

### 1.1 当前状态

Skill 功能的核心架构已完成，包括：
- 用户技能 CRUD、公共技能发现、分享审核流程
- 前端页面（技能管理、发现、管理员审核）
- Agent 集成（AI 驱动的技能选择模式）

### 1.2 发现的问题

| 优先级 | 问题 | 用户影响 |
|--------|------|----------|
| P0 | 技能使用统计未记录 | 统计页面无数据，无法分析技能效果 |
| P0 | 用户无法知道 AI 应用了哪个技能 | 黑盒体验，无法验证技能是否生效 |
| P1 | 用户技能无搜索功能 | 技能多时查找困难 |
| P1 | 无批量操作 | 管理多个技能效率低 |

1. **可观测性**：用户能清晰看到 AI 应用了哪个技能
2. **数据完整性**：技能使用统计正确记录
3. **易用性**：提升技能管理效率

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 技能应用可见率 | 100% | 每次技能应用都有显式标记 |
| 统计数据完整性 | 100% | skill_usage 表记录与实际应用一致 |
| 搜索使用率 | >30% | 有搜索行为的用户占比 |

## 2. 功能需求

### 2.1 P0-1: 技能应用显式标记

**用户故事**：作为用户，我希望在 AI 回复中看到它应用了哪个技能，以便验证技能是否生效。

**实现方案**：

在系统提示中添加指令，要求 AI 在应用技能时在回复开头标记：

```
当你决定应用某个技能时，在回复的第一行使用以下格式标记：
[使用技能: 技能名称]

然后空一行，开始正式回复。
```

**示例输出**：
```
[使用技能: 创建角色]

好的，让我帮你创建这个角色。首先我需要了解几个关键信息...
```

**验收标准**：
- [ ] AI 应用技能时，回复开头有 `[使用技能: xxx]` 标记
- [ ] 不应用技能时，无标记
- [ ] 标记格式统一，便于前端解析（可选高亮）

---

### 2.2 P0-2: 技能使用统计记录

**用户故事**：作为用户，我希望在统计页面看到技能使用数据，以便分析哪些技能最有效。

**实现方案**：

在 `stream_adapter.py` 中解析 AI 回复，检测 `[使用技能: xxx]` 标记，记录到 `skill_usage` 表。

**数据流**：
```
AI 回复 → StreamAdapter 检测标记 → 解析技能名称 → 查找技能 ID → 记录到 skill_usage
```

**记录字段**：
| 字段 | 来源 |
|------|------|
| skill_id | 根据技能名称查找 |
| skill_name | 从标记中解析 |
| skill_source | user / builtin |
| matched_trigger | "AI 自主选择" |
| confidence | 1.0 |
| user_message | 用户消息摘要 |

**验收标准**：
- [ ] 每次技能应用都记录到 skill_usage 表
- [ ] 统计页面正确显示使用数据
- [ ] 支持按时间范围筛选

---

### 2.3 P1-1: 用户技能搜索

**用户故事**：作为用户，我希望能搜索我的技能，以便快速找到需要的技能。

**实现方案**：

**后端**：
- `GET /api/v1/skills` 添加 `search` 查询参数
- 搜索范围：技能名称、描述

**前端**：
- 在"我的技能"标签页添加搜索框
- 实时过滤（防抖 300ms）

**验收标准**：
- [ ] 搜索框在技能列表上方
- [ ] 支持按名称和描述搜索
- [ ] 搜索结果实时更新
- [ ] 无结果时显示空状态

---

### 2.4 P1-2: 批量操作

**用户故事**：作为用户，我希望能批量启用/禁用技能，以便高效管理多个技能。

**实现方案**：

**后端**：
- `POST /api/v1/skills/batch-update` 批量更新接口

**前端**：
- 技能卡片添加复选框
- 选中后显示批量操作栏
- 支持操作：批量启用、批量禁用、批量删除

**验收标准**：
- [ ] 可多选技能
- [ ] 批量操作栏显示选中数量
- [ ] 批量删除需二次确认
- [ ] 操作完成后刷新列表

### 3.2 API 变更

**GET /api/v1/skills**
```diff
+ Query Parameters:
+   search: string (optional) - 搜索关键词
```

**POST /api/v1/skills/batch-update** (新增)
```json
{
  "skill_ids": ["id1", "id2"],
  "action": "enable" | "disable" | "delete"
}
```

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI 不遵循标记指令 | 标记不显示 | 在 prompt 中强调，必要时多次提醒 |
| 标记解析错误 | 统计数据不准 | 使用正则表达式严格匹配 |
| 批量删除误操作 | 数据丢失 | 二次确认 + 软删除 |
